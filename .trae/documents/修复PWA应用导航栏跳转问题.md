# 修复移动端标题与导航栏重叠问题

## 问题分析

* 导航栏使用 `position: fixed` 固定在顶部，z-index: 1000

* Hero 区域没有为固定导航栏留出足够的顶部空间

* 在移动端，导航栏可能会展开或占据更多空间，导致标题与导航栏重叠

## 解决方案

为 Hero 区域添加足够的顶部内边距，确保标题不会被固定导航栏遮挡。同时，在移动端添加专门的响应式样式来处理这个问题。

## 具体修改

1. 在 CSS 中为 `.hero` 类添加 `padding-top` 属性
2. 为移动端添加专门的响应式样式，确保在小屏幕设备上标题不会与导航栏重叠
3. 调整 `.hero-header` 的样式，确保标题在各种设备上都能正常显示

## 预期效果

* 在移动端打开时，标题不再与导航栏重叠

* 保持桌面端的布局美观

* 确保响应式设计在各种屏幕尺寸下都能正常工作
